# Тест-план интернет-магазина

## Цель тестирования

Проверить основной функционал интернет-магазина и убедиться, что пользователь может нормально пользоваться сайтом: искать товары, добавлять их в корзину и оформлять заказ.

## Что тестируется

В рамках проекта проверяются:

- авторизация пользователя
- поиск товара
- карточка товара
- корзина
- оформление заказа

## Основные сценарии

Проверяются следующие сценарии:

- вход с корректными данными
- вход с неверным паролем
- поиск существующего товара
- поиск несуществующего товара
- добавление товара в корзину
- удаление товара из корзины
- изменение количества товара
- оформление заказа
- оформление заказа с пустыми полями

## Типы тестирования

В проекте используются:

- функциональное тестирование
- smoke testing
- regression testing
- негативное тестирование
- UI testing

## Что не тестируется

В рамках проекта не проверяются:

- нагрузка на сайт
- безопасность платежей
- backend на уровне кода
- работа сайта во всех браузерах

## Тестовое окружение

- Windows 11
- Google Chrome
- Desktop

## Критерии завершения тестирования

Тестирование считается завершенным, если:

- выполнены основные проверки
- составлены тест-кейсы
- оформлены баг-репорты
- критические ошибки зафиксированы

## Возможные риски

- нестабильная работа сайта
- отсутствие тестовых данных
- неполные требования

## Итог

После тестирования должна быть подтверждена работоспособность основных пользовательских сценариев интернет-магазина.
