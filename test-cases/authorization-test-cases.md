# Тест-кейсы: авторизация

## TC-AUTH-01. Вход с корректными данными

**Приоритет:** High  
**Тип:** Positive

### Предусловия

Пользователь уже зарегистрирован.

### Тестовые данные

- Email: testuser@example.com
- Пароль: Password123

### Шаги

1. Открыть сайт интернет-магазина.
2. Нажать кнопку "Войти".
3. Ввести email.
4. Ввести пароль.
5. Нажать кнопку "Войти".

### Ожидаемый результат

Пользователь входит в аккаунт и попадает на главную страницу или в личный кабинет.

## TC-AUTH-02. Вход с неверным паролем

**Приоритет:** High  
**Тип:** Negative

### Предусловия

Пользователь уже зарегистрирован.

### Тестовые данные

- Email: testuser@example.com
- Пароль: WrongPassword123

### Шаги

1. Открыть сайт интернет-магазина.
2. Нажать кнопку "Войти".
3. Ввести корректный email.
4. Ввести неверный пароль.
5. Нажать кнопку "Войти".

### Ожидаемый результат

Пользователь не входит в аккаунт. На странице появляется сообщение об ошибке.

## TC-AUTH-03. Вход с пустым полем email

**Приоритет:** Medium  
**Тип:** Negative

### Шаги

1. Открыть страницу авторизации.
2. Оставить поле email пустым.
3. Ввести пароль.
4. Нажать кнопку "Войти".

### Ожидаемый результат

Пользователь не входит в аккаунт. Под полем email появляется сообщение, что поле нужно заполнить.

## TC-AUTH-04. Выход из аккаунта

**Приоритет:** Medium  
**Тип:** Positive

### Предусловия

Пользователь авторизован.

### Шаги

1. Открыть личный кабинет.
2. Нажать кнопку "Выйти".

### Ожидаемый результат

Пользователь выходит из аккаунта.
