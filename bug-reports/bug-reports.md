# Баг-репорты интернет-магазина

## BUG-01. Нет сообщения при поиске несуществующего товара

**Severity:** Medium  
**Priority:** Medium  
**Status:** Open

### Шаги воспроизведения

1. Открыть сайт магазина.
2. Ввести qwerty12345 в поиск.
3. Нажать кнопку поиска.

### Фактический результат

Открывается пустая страница без сообщения.

### Ожидаемый результат

Должно отображаться сообщение "Товары не найдены".

---

## BUG-02. Итоговая сумма не обновляется

**Severity:** High  
**Priority:** High  
**Status:** Open

### Шаги воспроизведения

1. Добавить товар в корзину.
2. Открыть корзину.
3. Изменить количество товара.

### Фактический результат

Количество товара меняется, итоговая сумма остается прежней.

### Ожидаемый результат

Итоговая сумма должна пересчитываться автоматически.

---

## BUG-03. Заказ оформляется с пустыми полями

**Severity:** Critical  
**Priority:** High  
**Status:** Open

### Шаги воспроизведения

1. Добавить товар в корзину.
2. Перейти к оформлению заказа.
3. Оставить обязательные поля пустыми.
4. Нажать кнопку "Подтвердить заказ".

### Фактический результат

Заказ успешно оформляется.

### Ожидаемый результат

Заказ не должен оформляться. Пользователь должен увидеть ошибки заполнения.

---

## BUG-04. Товар не удаляется из корзины сразу

**Severity:** Medium  
**Priority:** Medium  
**Status:** Open

### Шаги воспроизведения

1. Добавить товар в корзину.
2. Открыть корзину.
3. Нажать кнопку "Удалить".

### Фактический результат

Товар исчезает только после обновления страницы.

### Ожидаемый результат

Товар должен удаляться сразу.

---

## BUG-05. Некорректное сообщение при ошибке входа

**Severity:** Low  
**Priority:** Medium  
**Status:** Open

### Шаги воспроизведения

1. Открыть страницу входа.
2. Ввести правильный email.
3. Ввести неверный пароль.
4. Нажать кнопку "Войти".

### Фактический результат

Отображается сообщение "Error".

### Ожидаемый результат

Должно отображаться понятное сообщение, например "Неверный email или пароль".
